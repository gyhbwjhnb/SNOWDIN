# Framework

本项目定位为一个基于前端模拟命令行交互的个人信息站，用户进入网站后通过输入命令来获取个人信息、项目经历、技能介绍和联系方式。

## 推荐技术栈

- React 19
  作为核心 UI 框架，用于构建命令行界面、输出区域、页面内容区和交互逻辑。

- TypeScript
  用于约束命令系统、数据结构和组件状态，降低后续维护成本。

- Vite
  作为前端构建工具，提供更快的本地开发启动速度和更轻量的工程配置。

- React Router
  用于管理页面路由，并让命令输入和页面跳转之间建立映射关系，例如 `about`、`projects`、`contact` 等命令可直接对应不同页面。

- Zustand
  用于管理终端历史记录、当前输入内容、主题状态、命令执行结果等全局状态，简单直接，适合本项目。

- Tailwind CSS
  用于快速构建命令行风格界面，包括排版、间距、颜色、边框和响应式布局。

- Motion
  用于实现更自然的终端交互动画，例如字符逐步输出、光标闪烁、模块切换和内容出现动画。

## 内容组织建议

- 页面信息使用本地 `TS` 或 `JSON` 文件管理。
- 常见内容包括：
  - `about`
  - `projects`
  - `skills`
  - `contact`
  - `resume`
- 常见命令包括：
  - `help`
  - `about`
  - `projects`
  - `skills`
  - `contact`
  - `resume`
  - `clear`
  - `theme`

## 实现建议

- 不使用传统博客方案，项目重点不是文章发布，而是个人信息展示。
- 不强依赖 `xterm.js`，优先使用 React 自定义实现伪终端 UI，这样更轻量，也更容易控制视觉和交互。
- 页面层保留正常路由，命令系统作为访问入口，这样既保留命令行风格，也方便后续扩展和部署。

## 最终推荐组合

- React 19
- TypeScript
- Vite
- React Router
- Zustand
- Tailwind CSS
- Motion
